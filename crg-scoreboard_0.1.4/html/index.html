Use Google Chrome! Use Google Chrome! Use Google Chrome!

Get your attention? Ok, I may be a Google fanboi. But Google Chrome really is far better/faster at executing JavaScript than any other browser. For example try the SunSpider JavaScript Benchmark to compare browser JavaScript performance.

Since the (current) scoreboard display uses JavaScript fairly heavily, especially during animations (i.e. moving images/clocks around), I highly recommend using Google Chrome for (at least) displaying the scoreboard. For the control panels/pages, which are not really as JavaScript-heavy, you can be able to use any browser; but Google Chome works good on those too. I will warn you that if you are using a less-than-fast computer, the scoreboard animations (i.e. moving images/clocks around) will almost certainly be choppy if you use anything besides Google Chrome. Try it for yourself and see.

ScoreBoard operation with a single laptop

You need a single laptop (or desktop) computer, a projector, and a projection screen (or wall).

  1. Since you are reading this, you should have already downloaded and started the latest scoreboard code. If so, skip to the next step. If you do need to get the code:
    1. Download the latest zipfile from the project files page.
    2. Extract the files from the zipfile.
    3. If you don't already have Java 1.5 or later installed, download and install it. The latest Java JRE is recommended.
    4. Run the script for your system to start the scoreboard program. You only need to do ONE of the steps below:
      • On Windows, run (Open or double-click) the start-Windows.bat file.
      • On Linux/UNIX, run the start-Linux.sh file.
      • On Mac, run the start-MacOS.command file. (Since Mac OS X is a UNIX system, you could run the start-Linux.sh from a Terminal instead)
      • For those familiar with Java, you can run Java directly in whatever way you want. You need to include all JAR files in the lib/ directory, and the main class is com.carolinarollergirls.scoreboard.ScoreBoardManager
    5. Point your web browser at the scoreboard main index page. The recommended browser is the latest Firefox, but IE 8, Opera, and Safari should work. You only need to do ONE of the steps below:
      • Open the start.html file, which will redirect you to the scoreboard index page.
      • Open your browser to http://localhost:8000/
  2. Connect the laptop's video output to the projector. How to do this depends on your laptop and your projector.
  3. Configure the laptop displays:
    1. Configure the laptop's video output to the projector as your secondary screen, and your laptop display as your primary display. You can reverse them if you want, the important thing is to have the laptop screen and projector be separate displays. These instructions assume your projector is the secondary display.
    2. Set the laptop's secondary display resolution to 1024x768, or any other resolution that has a 4:3 aspect ratio (for example, 1024/768 = 4/3 = 1.3333...). Note that 1280x1024 is 5:4 aspect, not 4:3.
  4. You already have one browser window open to this page. Open a new browser window and go to the main scoreboard page.
  5. Drag the secondary browser window, showing the main scoreboard, over to the projector display.
  6. Put the secondary browser window into full-screen mode. On Firefox (which is highly recommended as the browser to use) you can do this by pressing F11.
  7. On the primary browser window, which is still pointing to these instructions, go to the single operator control panel. That panel provides basic functionality. You can return to this page at any time and the scoreboard will continue to operate.
  8. At any time, if you want to save the scoreboard to file, or load the scoreboard using a previously saved file, use this page.

BETA-level "UN"-controls

The one-operator control panel has new "UN"-controls that are, by default, hidden. You can show the controls by checking the box above in the Quick Links section and clicking on the Control Panel link.

The "UN"-controls allow you to "undo" an accidental "Start Jam", or "Stop Jam", or "Timeout", however they only control the affected clocks, and they do not prevent any Policies from running based on the clocks stopping/starting. Also you cannot undo more than one action, so (for example)you should not try to undo a Timeout and then immediately try to undo a StartJam. Again, the "UN"-controls are BETA level so please use them with that in mind.

Using an iPhone to control the scoreboard

While a dedicated iPhone application would be better, the operator control panel is designed to be usable from an iPhone. Just point the iPhone's browser at the page. Note that clicking the buttons from an iPhone is not quite as responsive as using an actual computer; for example adding 4 points requires slowly pressing the +1 point team button 4 times. Fast multiple clicks will not all register simply because the iPhone can't (or won't) handle all of the presses.

Managing Images

The main scoreboard uses images in several places; for the Team Logos, the scorboard background, intermission background, countdown background, etc. All the images can be viewed on the Image Management page. The images cannot be modified except for their Name attribute. Their name is used in the drop-down lists that allow selecting an image. The images are separated into different types, which allows selecting an image from only the correct type group. The current groups defined are ad, teamlogo, and fullscreen, although you can define any image group in the crg.scoreboard.properties file. For example, in the operator control page, the drop-down lists that allow setting each Team's Logo only list images that are of type teamlogo. Also, the main scoreboard's advertisement image that is displayed during Lineups automatically cycles randomly among all the images of type ad.

To remove images, the actual image file must be removed from the image type's directory. So for example if the teamlogo image type corresponds to the images/teamlogos directory, and you want to remove the carolina.png Team Logo image, you would need to remove (or move) the images/teamlogos/carolina.png file. Once you remove the file, go to the image management page and click the Update Images button at the top. Any images removed from your system will be completely removed from the scoreboard. Any teamlogo images that are in use by a team will clear that team's logo to no image.

To add images, just drop the image(s) into the appropriate image type directory. So with the same example as above, if you want to add the carolina.png Team Logo image as a teamlogo type image, and the teamlogo image type corresponds to the images/teamlogos directory, just drop the image file into that directory. Then click the Update Images button as above. By default, all new images use their file name as their Name. You can edit their Name after adding them. Edited image names will be saved as part of the scoreboard XML file (when saving from the save/load page), but only for those images that currently exist (i.e., once you remove an image any edited Name is lost).

Images can be any type or size, but the current main scoreboard page uses specific aspect ratios. Use these suggested image sizes (or at least stay close to the same aspect ratio) if you are displaying the images with the main scoreboard page.

  • Team Logo images should be 640x240, or aspect ratio 2.667:1
  • Fullscreen images should be 1600x1200, or aspect ratio 4:3 (the main scoreboard is fixed at 4:3 ratio and resizes to fit the browser window)
  • Ad images should be 1600x320, or aspect ratio 5:1

Configuration files

The main configuration file is the crg.scoreboard.properties Java Properties file. It is located (by default) in the lib directory.

The second configuration file is the crg.scoreboard.xml file, also located (by default) in the lib directory. It contains the default XML settings that are loaded into the scoreboard on startup.

Settings in the crg.scoreboard.properties file

The default file itself has documentation as to what each property is. See the file itself for more details. For any property ending in .N, it means multiples of that property can be defined, and for each one replace the N with a unique identifier, such as sequential numbers. Some properties use the identifier, and document what its meaning is.
